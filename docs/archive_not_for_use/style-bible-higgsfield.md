# Стайл-байбл The Lost Silence — для Higgsfield

Справочник для генерации персонажей, локаций и видео в Higgsfield. Собран из уже существующего
канона проекта (`TLS_Lore_v2.docx`, `NPC_TLS_v2.docx`, `Flora and Fauna TLS.docx`, цвета фракций
из `src/core/seed.ts`) — почти всё здесь уже было решено тобой раньше, просто не в одном месте.

Главный вывод из разбора папки `flux_nomads`: расхождение шло не от того, что стиля не было —
`NPC_TLS_v2.docx` уже содержит по промпту на каждого NPC каждой фракции (см. раздел 7). Проблема
в том, что реальные картинки в `full_size` этим промптам почти не следовали (Марек — не
путеводитель с компасом, а фэнтези-маг с нимбом; Лиа — не инженер в комбинезоне, а боевая
броня с пушками). Этот документ — рамка, которая не даст повторить это на следующих 5 фракциях.

---

## 0. Как этим пользоваться

Каждая генерация в Higgsfield собирается из четырёх слоёв, всегда в этом порядке:

1. **Глобальный рендер-стандарт** (раздел 1) — одна и та же техническая база для вообще всех
   изображений проекта, независимо от фракции.
2. **Фракционный модуль** (раздел 3) — цвет, материалы, культура одежды конкретной фракции.
3. **Промпт персонажа** — либо уже готовый из `NPC_TLS_v2.docx` (для расписанных NPC), либо
   написанный по шаблону раздела 2.3 (для новых лиц).
4. **Негативный промпт** (раздел 6) — вставляется всегда, без исключений.

Если генерация «не туда съехала» — почти всегда проблема в том, что пропущен слой 1 или 4:
модель по умолчанию тянет в сторону дженерик fantasy/sci-fi иллюстрации, если явно не сказать,
чего не должно быть в кадре.

## 1. Глобальный рендер-стандарт (для всех фракций)

Это ядро промпта, которое не меняется от фракции к фракции — меняется только модуль раздела 3.

**Формат кадра:**
- Full-body portrait, transparent alpha background (для карточек NPC/инвентаря — герой должен
  вырезаться из фона без обрезанных рук/ног);
- для окружения/локаций — кадр со средой, но тот же рендер-движок и грейд (раздел 4).

**Стиль рендера:**
- semi-realistic digital painting — не фотореализм, не аниме, не «глянцевая фэнтези-обложка»,
  не 3D-рендер игры;
- мягкий, мотивированный свет (источник света должен быть понятен — окно, костёр, экран,
  небо — а не абстрактный студийный рим-лайт со всех сторон);
- камера: full-body, eye-level, эквивалент 35–50 мм (без рыбьего глаза, без драматичных ракурсов
  снизу вверх — мир не героизирует персонажей);
- выражение лица: сдержанное, человеческое — спокойствие, усталость, лёгкая ирония, тревога.
  Никакого «эпического» прищура фэнтези-героя.

**Почему это важно:** по лору мир 2670+ выглядит рационально и буднично снаружи (см. `Том I`:
«мир выглядит здоровым, чистым и рациональным»), а не как space opera. Персонажи — рабочие
люди при исполнении обязанностей, а не воины/маги. Это касается вообще всех шести фракций,
даже самых технологичных (Aeralis, Hydrosynth) — технологии там встроены в быт, а не выставлены
напоказ как оружие или магия.

### 1.1. Базовый шаблон техстроки (вставлять в конец почти любого промпта)

```
Full-body portrait, transparent alpha background, semi-realistic digital painting,
soft motivated lighting, eye-level camera, 35–50mm, restrained natural expression.
```

## 2. Работа с Higgsfield: Soul ID, Moodboard, модели

*(на основе актуальной документации Higgsfield, июль 2026)*

### 2.1. Soul ID — консистентность лица персонажа
Soul ID (модель Soul 2.0) фиксирует лицо, стиль и идентичность персонажа между генерациями —
это то, что не даёт лицу «плыть» от карточки к карточке. Работает по загруженному набору
изображений одного лица (в идеале 20+, с разных ракурсов, при похожем освещении).

Практика для проекта:
1. Сгенерировать 1 «эталонный» портрет персонажа по промпту из `NPC_TLS_v2.docx` +
   фракционный модуль + негативный промпт — пока не получится лицо, которое устраивает.
2. Нагенерировать 5–10 вариаций этого же промпта с небольшими изменениями ракурса/света —
   это и есть «набор фото» для Soul ID.
3. Создать Soul ID персонажа из этого набора. Дальше все его дальнейшие сцены (диалоговые
   портреты, боевые карточки, полноростовые для карты) генерируются через этот Soul ID —
   лицо и силуэт больше не «уплывают».

### 2.2. Moodboard — фиксация фракционного стиля
Moodboard в Soul 2.0 — кастомный style-preset, собранный из своих референсов (в отличие от
20+ встроенных пресетов вроде editorial/high-fashion). Это ключевой инструмент для проекта:

- Собери на каждую фракцию мудборд из 4–8 картинок: 1–2 лучших существующих персонажа этой
  фракции (или чужие референсы одежды/материалов похожего духа) + 1–2 референса архитектуры/
  окружения фракции + цветовая плашка фракции (раздел 3).
- Сохрани как персональный style-preset фракции. Используй его вместо (или поверх) общих
  пресетов Higgsfield при каждой генерации персонажа/сцены этой фракции.
- Это единственный надёжный способ удержать «дух» фракции стабильным на 15+ NPC каждой —
  промпт-слова одни и те же, но без мудборда модель всё равно будет тянуть в сторону
  дженерик fantasy/sci-fi по своим весам.

### 2.3. Шаблон промпта для нового персонажа
Если персонажа ещё нет в `NPC_TLS_v2.docx`, пиши по той же структуре, что уже используется
там (это тот же формат, что доказал свою работоспособность для готовых NPC):

```
Full-body [semi-realistic] portrait, alpha background.
[Имя] — [роль/профессия]. [этничность/микс, возраст], [1–2 черты лица/фигуры].
[Верхняя одежда: материал + фракционная деталь техвещи]. [1–2 аксессуара, привязанных к профессии].
Colors: [2–3 цвета из фракционной палитры, раздел 3].
Pose: [нейтральная поза при исполнении]. Expression: [сдержанное].
```

### 2.4. Видео (Higgsfield video-модели)
Higgsfield даёт доступ к нескольким видео-моделям с разными типами motion control (в духе
Kling/Seedance — управление траекторией камеры и объектов). Для проекта:
- Стартовый кадр для видео — всегда сначала зафиксированный Soul ID-портрет/сцена, а не
  промпт с нуля: так лицо и палитра не разъезжаются между первым и последним кадром.
- Движение камеры подбирается по характеру фракции (раздел 5), а не берётся дефолтным —
  дефолтные «кинематографичные» движения Higgsfield почти всегда читаются как
  space-opera-трейлер, что для этого мира не подходит нигде, кроме, может быть, Aeralis.

## 3. Фракционные модули

Канонические hex-цвета — из `src/core/seed.ts` (уже используются в UI игры, значит должны
совпадать с художкой). Материал/border — из `boxStyle`, переведён в художественный язык.
Одежда/техника — словарь, реально уже использованный в готовых NPC-промптах фракции
(`NPC_TLS_v2.docx`) — не изобретать новый, продолжать этот же.

### 3.1. Flux Nomads
- **Канон-цвет:** `#4fd1c5` (бирюзово-тил). Акцент, не основа — основа одежды приземлённая.
- **Материал (boxStyle: pulse):** «дыхание генераторов» — ровный, рабочий, слегка вибрирующий
  тех-акцент (индикаторы, подсветка на рукаве), никакого глянца.
- **Мир:** рабочий мегагород, грубый, шумный, многоуровневый, в постоянном ремонте — «не витрина
  цивилизации» (Досье B). Мобильные лагеря — временные, функциональные, без орнамента.
- **Одежда/словарь:** layered utility jacket, tool harness/rig, cargo pants, reinforced boots,
  belt with pouches/scanners, longcoat with signal gear. Палитра: burnt orange, charcoal gray,
  muted sand, dusty blue, olive, rust red — **никогда** чистый монохром без потёртости.
- **Do:** практичная многослойная рабочая одежда; видимые инструменты профессии (сканер,
  паяльник, датапад, компас); лёгкая изношенность (пятна, потёртости, пыль).
- **Don't:** глянцевая боевая броня/экзоскелет как у солдата (это не наёмники); плащи и
  мантии волшебников; парадная/викторианская одежда; космический мостик/голограммы как фон
  (Flux Nomads — про пустыню и палаточный лагерь, не про звездолёт).
- **Животное-символ:** песчаный скат — планирует над землёй, никогда не останавливается.

### 3.2. Sylvarium
- **Канон-цвет:** `#98c379` (живой зелёный).
- **Материал (boxStyle: shimmer, radius 20):** текучая, «живая» линия — мягкие обводы,
  никаких острых тех-граней.
- **Мир:** биоинженерные рощи, светящиеся пыльцевые организмы, органическая архитектура,
  вросшая в живое дерево. Технологично, но выглядит первобытно (Досье C).
- **Одежда/словарь:** robes / woven clothing с вросшими живыми ветвями, herb satchels,
  bio-honeycomb modules, glowing pollen-lanterns, soft biolamp glow. Палитра: зелёно-жёлтый
  природный свет, землистые тона, мягкое био-свечение (никогда неоновое).
- **Do:** живые/органические детали как часть костюма (не украшение); мягкий природный или
  биолюминесцентный свет; спокойные, «слушающие» позы.
- **Don't:** чёрный цвет в трауре/горе (канон: «горе тихое, без чёрного цвета» — смерть у
  Sylvarium это компост, а не могила); жёсткий металл/сталь как основной материал; резкий
  индустриальный свет.
- **Животное-символ:** не закреплено в каноне — при необходимости предлагать, а не придумывать
  волевым решением (уточнить у владельца, если понадобится для контента).

### 3.3. Woodhaven
- **Канон-цвет:** `#d19a66` (тёплая охра/дерево).
- **Материал:** сознательно анти-глянцевый (единственная фракция без `spatial`-поверхности
  в UI) — матовые, тканевые, деревянные текстуры, никакого «технологичного» блеска даже там,
  где есть техника.
- **Мир:** малые поселения, ручной труд, тёплый, домашний тон с тяжёлым наследием катастрофы
  (сама формулировка канона — «тон: тёплый, домашний, но с тяжёлым наследием»).
- **Одежда/словарь:** leather aprons, woven shawls with symbolic embroidery, wood-carved tools/
  talismans, hybrid old-tech (например, «glowing forge hammer» — единственный принятый
  тех-акцент, кузнечный, не цифровой). Палитра: earth-tone, warm brown, muted rust, natural
  fabric white.
- **Do:** ручной труд в кадре (инструмент, а не гаджет); плетёные/вязаные узоры; естественный
  дневной свет, тёплый тон заката/очага.
- **Don't:** голографические интерфейсы, гладкий пластик/металл, неон, что-либо, что выглядит
  «городским» или заводским.
- **Животное-символ:** великий олень — древний, охраняемый, символ хранителя лесов.

### 3.4. Cavernium
- **Канон-цвет:** `#b39cf0` (лавандово-фиолетовый).
- **Материал (boxStyle: glass 4, radius 6):** почти без «стекла» — плотный, тяжёлый, минимум
  прозрачности и мягкости, углы менее скруглены, чем у всех остальных.
- **Мир:** подземные мегаполисы под давлением, дисциплина и иерархия, минералы и кристаллизованные
  фрагменты сознаний погибших после катастрофы (Досье E + `NPC_TLS_v2.docx` интро) — это не
  просто шахта, у камня в этом мире есть «память».
- **Одежда/словарь:** heavy mining exosuits, reinforced plates, oil/mineral-dust stains, welding
  sparks, glowing crystal shards embedded in gauntlets/tools (фиолетовое свечение минералов —
  единственный «магический на вид» элемент фракции, и он обоснован лором, не случаен).
  Палитра: тёмный камень, графит, ржавчина металла + точечный фиолетовый/минеральный glow.
- **Do:** тяжёлая индустриальная защита; пыль и грязь как часть текстуры; тёмный фон, точечный
  свет фонаря/кристалла, а не общее освещение сцены.
- **Don't:** лёгкая ткань/шёлк как основа костюма; яркое равномерное освещение (Cavernium никогда
  не выглядит «солнечно»); открытые пространства на фоне.
- **Животное-символ:** каменный медведь — медленный, огромный, практически неуязвимый.

### 3.5. Aeralis
- **Канон-цвет:** `#7db8f0` (небесно-голубой).
- **Материал (boxStyle: glass 30, radius 22):** максимум «воздуха» и прозрачности из всех
  фракций — высота, лёгкость, минимум визуального веса.
- **Мир:** стратосферные платформы, орбитальные узлы, климат-контроль; дистанция и наблюдение,
  минимальное вмешательство в наземные дела (Досье F).
- **Одежда/словарь:** aerotech mantles/suits с wind-control filaments, winged harness, floating
  micro-sensors/spheres вокруг персонажа, holo-blueprint/atmospheric map panels. Палитра:
  белый, светло-серый, небесно-голубой; свет — «soft overhead sky lighting», отражения облаков.
- **Do:** ощущение высоты и простора даже в портрете (светлый фон, воздух вокруг фигуры);
  аэродинамичные силуэты одежды; сдержанная, «наблюдающая» поза.
- **Don't:** тяжёлая грузная броня (это не Cavernium); тёмная/грязная палитра; замкнутые
  тесные интерьеры как фон.
- **Животное-символ:** небесный парусник — гигантское воздушное существо, символ свободы.

### 3.6. Hydrosynth
- **Канон-цвет:** `#2ec4d6` (глубокий бирюзово-голубой — отличать от более зеленоватого тила
  Flux Nomads, у Hydrosynth цвет холоднее и «мокрее»).
- **Материал (boxStyle: shimmer, glass 20, radius 20):** свет в толще воды — текучий, мягко
  преломлённый, полупрозрачный, но не такой воздушный, как у Aeralis (плотнее, глубже).
- **Мир:** подводные города-станции, распределённая инфраструктура, работа с давлением и
  изоляцией (Досье G).
- **Одежда/словарь:** flowing aqua-blue coats/robes, hydrotech filaments/conduits вдоль рукавов,
  diving suits, liquid-glass tools, bioluminescent reflections на коже/ткани, glowing marine
  holograms. Палитра: аква-синий, бирюза, тёмная морская синева, точечный биолюм-glow.
- **Do:** влажный/мокрый блеск материала (не сухой глянец); мягкое голубое свечение снизу или
  сбоку, как будто отражение воды; текучие, а не резкие линии кроя.
- **Don't:** сухая пыльная палитра (это Cavernium/Flux); резкий верхний студийный свет;
  угловатые жёсткие силуэты.
- **Животное-символ:** лунный кит — гигантское морское существо, чьи «песни» слышны издалека.

## 4. Локации и окружение

Общие анкеры мира (из `Flora and Fauna TLS.docx`) — использовать в любом фоне/локации, вне
зависимости от фракции:
- Природа не враг и не угроза — она безразлична и по умолчанию мирная. Не рисовать лес/болото/
  джунгли как источник опасности по умолчанию.
- Масштаб: леса 150–200 м высотой; небоскрёбы XXI века стоят внутри них, как «молодые деревца
  среди исполинов» — если в кадре есть руины старого мира, они должны выглядеть маленькими и
  обрастающими, а не главным элементом кадра.
- Три источника флоры/фауны в кадре: наследие (обычные волки/берёзы/мхи — для эмоционального
  моста с XXI веком), архогенез (светолоза, пыльцевые медузы, полировочные деревья — новые виды,
  не мутанты), симбионты (навигаторы-птицы, дождевые киты — Mesh + природа вместе). Не смешивать
  с «фэнтези-магией» — всё объясняется биотехом, не волшебством.

Архитектурный язык — по фракции (раздел 3): рабочий мегагород (Nomads) / органические рощи-
узлы (Sylvarium) / малые ручные поселения (Woodhaven) / плотные подземные мегаполисы (Cavernium)
/ стратосферные лёгкие платформы (Aeralis) / распределённые подводные станции (Hydrosynth).
Nexus как фон — отдельно: стерильный, «scan»-эстетика, машина сглаживания реальности, а не жилой
город (используется как контраст к фракциям, а не как седьмая фракция для этого документа).

## 5. Видео — движение камеры по фракциям

| Фракция | Характер движения |
|---|---|
| Flux Nomads | ручная, документальная камера — лёгкая тряска, практичные панорамы, как будто снято на ходу |
| Sylvarium | медленный органический дрейф, как будто камера сама «дышит» вместе с лесом |
| Woodhaven | статичная или очень медленная камера, тёплая, без спешки |
| Cavernium | тяжёлая, статичная, редкие резкие движения (авария/тревога) — вес и давление |
| Aeralis | плавные широкие скольжения, дроновые облёты, ощущение высоты |
| Hydrosynth | медленный подводный снос по течению, лёгкое покачивание |

Правило: не брать дефолтные «кинематографичные» траектории Higgsfield без проверки — они почти
всегда читаются как трейлер космооперы, что не подходит ни одной из фракций, кроме отдельных
сцен Aeralis.

## 6. Негативный промпт (мастер-список)

Вставлять во все генерации персонажей/сцен проекта:

```
fantasy magic robes, glowing magic orbs, wizard/sorcerer aesthetic, ornate ceremonial armor,
full battle-mech power armor, Victorian/period formal wear, spaceship bridge backdrop,
```
