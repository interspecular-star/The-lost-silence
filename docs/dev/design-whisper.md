# Канал Архона — «Шёпот» (дизайн, H3)

Статус: НА УТВЕРЖДЕНИИ у владельца. Код не пишется, пока док не согласован.
Контекст: решения владельца 2026-07-09 — голос в голове не должен прерывать игру;
блок от края до края под HUD; двухуровневое общение (клик по полосе → чипы-ответы).
Канон: `oskolok-mesh.md` — Mesh = голос Архона, выключатель у игрока.

---

## 1. Принцип

**Шёпот — не диалог, а событие.** Игра не останавливается: сцена кликабельна,
бой идёт, диалоги с людьми продолжаются. Голос живёт в собственном слое.

Два режима голоса:
- **Шёпот** (этот док) — фоновый, необязательный, 95% случаев.
- **Вмешательство** — редкие сюжетные точки: обычный полноэкранный диалог
  (уже существующая механика) с материалом «Архон». Новый код не нужен.

## 2. Вид и поведение полосы

```
┌──────────────────────────────────────────────────┐
│ HUD: ◈ ур/hp/foc …                    ⌬ кредиты │
│──────────────────────────────────────────────────│
│ ◈  Ты опять не спишь. Люди Кая считают тебя…    │ ← полоса шёпота
│ ~~~~~~~~ спектральная линия ~~~~~~~~~~~~~~~~~~~~ │
└──────────────────────────────────────────────────┘
```

- От края до края, без скруглений, под HUD. Стекло плотнее обычного
  (glass ~6%), сверху границы нет, снизу — hairline с рамкой «Спектр».
- Слева пульсирует глиф ◈. Цвет — не фракционный (спектральный дрейф).
- Текст **печатается** (`[type]`), скорость ~35 мс/знак.
- Появление: полоса выдвигается сверху за ~250 мс. НЕ гасит сцену.
- Материал полосы настраивается (пресет «Архон» из будущей библиотеки H2;
  до H2 — захардкоженный дефолт с ручкой в теме).

Жизненный цикл:
1. Появилась, текст напечатался.
2. Висит `holdSec` (по умолчанию 6 с; на длинный текст — авто-добавка).
3. Игрок НЕ тронул → тает, сворачивается в глиф ◈ возле HUD (маленький,
   полупрозрачный). Счётчик `mesh_ignored` +1, если шёпот ждал ответа.
4. Игрок кликнул полосу (или хоткей `Q`) → полоса раскрывается вниз на
   один ряд: 1–3 **чипа-ответа** (короткие пилюли). Выбор чипа → эффекты
   (mesh_trust и т.п.), опциональный ответ Архона следующим шёпотом.
5. Клик по глифу ◈ → перечитать последний шёпот (без чипов, если истёк).

## 3. Правила вежливости

- Один шёпот на экране; остальные — в очередь (FIFO, максимум 3,
  лишние тихо отбрасываются — Архон не бубнит).
- В бою: не появляется во время замаха/QTE; ждёт паузу между ходами.
- Поверх диалога с NPC — МОЖНО (комментарий о собеседнике), но полоса
  приглушена на 20%, чтобы не спорить с репликой человека.
- `mesh_on = false` → канал мёртв полностью. Никаких «пропущенных».
- Один и тот же шёпот не повторяется (сработал — заснул навсегда),
  если не помечен `repeatable` (для «праздных приветов» с кулдауном).
- reduced-motion: без печати, текст появляется сразу.

## 4. Молчание = ответ

- `mesh_ignored` (число) — авто-переменная: сколько шёпотов с чипами
  игрок оставил без ответа. Доступна в условиях, как любая переменная →
  Архон реагирует контентно («Ты снова молчишь…»), без спец-кода.
- `mesh_answered` — сколько раз ответил. Оба видны в отладке предпросмотра.

## 5. Журнал шёпота

Вкладка «◈ Голос» в журнале (📋): хронология всех показанных шёпотов
(текст + выбранный чип или «— промолчал»). Хранится в сейве (последние ~50).

## 6. Данные (набросок)

```
WhisperDef {
  id, text (с разметкой textfx),
  conditions: Condition[]          // когда МОЖЕТ прозвучать
  trigger: 'enterScene' | 'varChanged' | 'dialogueEnd' | 'combatEnd'
         | 'idle' | 'manual'       // manual — только эффектом из диалога
  sceneId?                         // для enterScene — конкретная сцена или любая
  delaySec?, holdSec?, repeatable?, cooldownMin?
  chips?: { text, effects, replyWhisperId? }[]   // 0–3
  priority?: 'normal' | 'important'  // important лезет вперёд очереди
}
project.whispers: WhisperDef[]
Эффект нового типа «прошептать» (whisperId) — из любой set-ноды/выбора.
Сейв: показанные id, mesh_ignored/answered, журнал.
```

## 7. Редактор

Секция «◈ Шёпот Архона» в режиме «Журнал» (рядом с заданиями — та же
логика «условия → событие»): список карточек, у каждой текст (панель
стилей textfx), триггер, условия (condsMini), чипы с эффектами
(effectsMini), тайминги. Кнопка «▶ проверить» — показать шёпот в
предпросмотре немедленно.

## 8. Этапы реализации (после утверждения)

1. H3a: слой полосы + очередь + печать + глиф + сейв/журнал (без чипов).
2. H3b: чипы-ответы, mesh_ignored/answered, хоткей Q.
3. H3c: редактор шёпотов + валидатор (битые ссылки, недостижимые шёпоты).
4. H3d: интро-контент: первые шёпоты после получения Осколка (глава 1).
