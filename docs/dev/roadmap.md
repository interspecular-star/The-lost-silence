  `engine.capturePlaytest()`.

  (`src/editor/validate.ts`): битые ссылки (сцены/диалоги/ноды/переменные/предметы/мобы/NPC),
  эффекты на computed, недостижимые ноды/сцены/диалоги, опечатки в `{подстановках}`,
  дубли имён переменных. Клик по проблеме — переход к месту.
- 💡 **A3. Поиск по проекту.** Где используется переменная / предмет / NPC / моб;
  переход по клику к месту использования.
- ✅ **A4. Цепочки квестов** (2026-07-03). `QuestStep[]` у задания (текст + условия);
  движок фиксирует пройденные этапы навсегда (`questSteps` в сейве/чекпоинтах),
  уведомления «📋 этап ✓»; в журнале список этапов (✓ / ▸ текущий / · будущие);
  редактор этапов в режиме «Журнал»; демо — «Наладить связи» в seed.
- 💡 **A5. Шаблоны и дублирование.** Дублировать моба/диалог/квест как заготовку;
  общие таблицы лута.
- ✅ **A6. Кинематографичные переходы** (2026-07-10, запрос владельца для смены глав).
  `ElementFx` (`SceneElement.fx`): появление fade/blur/rise/zoom с задержкой и
  длительностью + исчезновение через N секунд (`src/runtime/elementfx.ts`, таймеры
  чистятся при пересборке сцены). `Scene.autoNext` — автопереход по таймеру
  (диалог/бой не рвёт — ждёт), `Scene.fadeSec` — длительность кросс-фейда при уходе
  (1.5–2с для смен глав). Инспектор: секции «Появление / исчезновение» (элемент)
  и «Переход дальше» (сцена). Валидатор: битый автопереход + достижимость сцен.
  Рецепт титула главы: тёмная страница → текст fx blur-in (задержка) + «исчезнуть
  через N» → autoNext на первую сцену главы с fadeSec 1.5.
