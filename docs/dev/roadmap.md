## F. Внешний вид и авторский воркфлоу (2026-07-04, владелец расставил порядок)

Порядок владельца: ~~Персонажи~~ → авторский воркфлоу → ачивки → анимации. Персонажи
(глубокие профили NPC) уже сделаны выше. Дальше — по одному блоку за раз.

- ✅ **F1. Авторский воркфлоу для диалогов** (2026-07-05). Режим «Черновик»
  (`src/editor/draft.ts`, пункт «📝 Черновик» в меню диалога рядом с «⧉ Дублировать» —
  дублирование ЦЕЛОГО диалога уже было реализовано раньше, `sidebar.ts`): один большой
  textarea на весь диалог в формате «Спикер: текст» / «> вариант ответа», кнопка
  строит/обновляет линейную цепочку line/choice нод (`analyzeDraftableChain` +
  `applyDraftToDialogue`). При повторном открытии текст генерируется ОБРАТНО из нод —
  можно дополнять уже написанное. Ветвления/условия/эффекты (branch/jump/set,
  расходящиеся варианты) черновик не показывает и не трогает — только предупреждает
  перед обновлением, что делать с ними нужно по-прежнему в графе. Плюс Ctrl+D для
  ОДНОЙ ноды диалога (`graph.ts: duplicateSelectedNode()`, ремап id вариантов).
- ✅ **F2. Ачивки** (2026-07-05). `AchievementDef` (`types.ts`): название, описание,
  иконка-эмодзи, условия, необязательная награда (эффекты/предметы). `engine.achievements`
  (id → разблокировано навсегда) — по образцу `questSteps`: `checkAchievements()` проверяет
  условия в тех же точках, что `checkQuestSteps()` (старт, `applyEffects`, тик, знакомство
  с NPC), разблокировка необратима, даже если условие потом перестанет выполняться.
  Награда выдаётся автоматически (без «забрать» в журнале) + уведомление `🏆` (цвет
  `#f4d35e`, отличается от квестового `📋`). Вкладка «🏆 Достижения» в журнале (`journal.ts`,
  по образцу вкладки «Персонажи»); редактор — 4-я секция в режиме «Журнал» (`quests.ts`),
  переиспользует `condsMini`/`effectsMini`/`grantsMini`. Сохраняется в сейве и чекпоинтах
  плейтеста.
- ✅ **F3. Анимации-трюки** (2026-07-05). Третий fx-файл по образцу `bgfx.ts`/
  `dialoguefx.ts` — `src/runtime/uifx.ts`, `ensureUiFxStyles()`. Три эффекта:
  переливающийся диагональный блик на легендарных/архонт-предметах в инвентаре и
  экипировке (`addRarityShine()`, только 2 старшие редкости — остальным не нужно);
  одноразовый блик-проход у варианта ответа при наведении (`.dchoice-sheen`, не
  зацикленный, переигрывается при каждом hover); праздничный баннер по центру-сверху
  на разблокировку ачивки (`notifyAchievement()` — золотая рамка, пружинный вход, блик,
